RapidRoster

Turn a participant list into fair, rule-aware groups — teams, cohorts, shifts, classrooms, and more.

Get started

How it works

  1. upload_file Configure data

    Import a CSV, map column types (text, numeric, dates, times), and set how many groups you need.

  2. tune Configure goals

    Add Cluster, Separate, or Limit rules. Set priority and mark critical ones as hard constraints.

  3. auto_awesome Generate groups

    A primary run builds and improves groups. Secondary runs offer alternative layouts to compare.

  4. download Review and export

    Compare the primary result and any alternatives, then download your groupings as a CSV.

Why I built this

Manually building rosters is slow and easy to get wrong.

Teammate requests, availability overlap, role caps, and “keep these people apart” all fight each other in a spreadsheet.

RapidRoster lets organizers state those tradeoffs as goals, generate groups, then export and tweak until it feels right.

Three goals

Compose groupings from a few fundamental rules — not dozens of one-off settings:

  • group_work Cluster

    Keep similar participants together.

    Examples: siblings, departments, overlapping availability, teammate requests.

  • call_split Separate

    Spread similar participants apart.

    Examples: schools, experienced players, genders across teams.

  • filter_alt Limit

    Cap or require counts per group.

    Examples: maximum 2 goalkeepers, minimum 1 coach.

Each goal targets the columns you choose, can filter who it applies to, and uses exact or partial matching where needed. Soft goals guide the search by score; hard constraints never allow illegal placements.
