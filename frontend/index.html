RapidRoster

Grouping and scheduling made easy.

Get started

How it works

  1. Configure data — Import participants from a CSV. Map columns to types (text, numeric, dates, times, ranges, IDs). Cells can hold multiple values separated by semicolons. Set how many groups you need.
  2. Configure goals — Build rules with Cluster, Separate, or Limit. Set priority and mark critical rules as hard constraints.
  3. Generate groups — A primary run seeds groups, then improves them with the best relocate or swap each step. Secondary runs start from random layouts to offer alternatives.
  4. Review and export — Inspect the result, compare alternatives, and download groupings as a perfectly formatted CSV.

Why I built this

Manually building rosters is slow and easy to get wrong: teammate requests, availability overlap, role caps, and “keep these people apart” all fight each other in a spreadsheet. RapidRoster is built so organizers can state those tradeoffs as goals and let the generator work through them. Then export and tweak the results in a spreadsheet until you're happy with the results.

Three goals, not dozens of rules

You can compose groupings from setting a few fundamental goals:

  • Cluster — Keep similar participants together (siblings, departments, overlapping availability, teammate requests).
  • Separate — Spread similar participants apart (schools, experienced players, genders across teams).
  • Limit — Cap or require counts per group (maximum 2 goalkeepers, minimum 1 coach).

Each goal targets columns you choose, can filter who it applies to, and uses exact or partial matching where needed. Soft goals guide the search by score. Hard constraints never allow illegal placements — if they cannot be satisfied, you get a clear error.
